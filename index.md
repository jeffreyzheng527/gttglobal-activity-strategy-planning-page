# 01 战略定位
## 重新定义核心价值

**认知基座：**
我们做的不是「卖产品」，而是**「提高分销商成交率」**。

**核心优势：**
我们最大的优势不是“有多少产品”，而是**“最早知道客户行程”**。
* ✅ 确定已出票
* ✅ 确定时间
* ✅ 确定目的地

### 旅游链路对比与 KPI 重定义

| 平台类型 | 状态 | 痛点/优势 |
| :--- | :--- | :--- |
| **OTA 平台** | 知道意图 | ❌ 不确定是否成行 |

| **我们** | **已出票** | **✅ 确定性最高** |

**KPI 转变：**
* ~~我卖了多少个景点门票~~
* **我帮分销商：更容易成交 / 更专业 / 更省时间**

# 02 分销商画像
## 不是「是谁」，而是「怎么卖」

我们不需要 100% 准确的画像，但需要关注以下决策维度：

* **客源结构**：家庭客 / 老年团 / 学生 / 商务
* **语言文化**：华人社区 / 拉美裔 / 越南裔
* **决策方式**：旅行社强推荐 vs 客人自己决定
* **价格敏感度**：经常压价 vs 注重服务
* **产品复杂度**：只卖经典款 vs 可卖小众体验

# 03 推荐逻辑
## 从海量到精选

不要做复杂的“最佳推荐”，要做**“足够好 + 不踩雷”**。

**三层过滤漏斗：**

1.  **强规则过滤 (1000 → 50)**
    * 时间、语言、季节性、Top 转化
2.  **画像加权 (50 → 20)**
    * 价格匹配、人群适配、复杂度
3.  **人工精选 (20 → 6-10)**
    * **最终输出：每个目的地固定 6-10 个主推 SKU**

# 04 触达节点
## 推荐发生在哪个环节？

**阶段 1：机票预订中（存在感）**
* ❌ 不要直接卖，不要跳转
* ✅ 植入心智：“我能帮你把地面也一起解决”

**阶段 2：出票后（核心 B2B2C）**
* 核心：给旅行社一个**“可以直接转发给客人”**的东西
* 形式：白标页面、精选产品、价格清晰

**阶段 3：目的地/临近出发（捡漏）**
* 现实：客人不会装 App
* 解法：轻触达（WhatsApp/短信）+ 无需注册的 Web 页

# 05 渠道现实
## 接受 DOS/Call Line 的现实

**策略：不是把人拉到网站，而是把能力塞进流程。**

1.  **半自动工具 (MVP)**
    * Call Line 接单 → 系统自动生成“巴黎推荐包” → 销售一键发送。
2.  **极简 B2B 门户 (中期)**
    * 不是完整 OTA，而是只看推荐、只下单精选产品。
3.  **战略融合 (长期)**
    * 处理渠道冲突，重构销售体系。

# 06 技术实现
## SPA (单页应用) 技术

从“推销广告”转变为**“专属服务”**。

* **传统做法**：`www.site.com/tours`
    * 体验：花花绿绿的广告页
* **SPA 做法**：`site.com/my-trip?order=AF123`
    * 体验：**“这是专门为我准备的行程单”**

**动态渲染：**
前端抓取订单号 → 后台分析画像 → **瞬间重组页面结构**（商务客推米其林，家庭客推乐园）。

### H5 行程管家的生命周期

| 阶段 | 触发时机 | 客户心理 | 展示重点 |
| :--- | :--- | :--- | :--- |
| **Pre-trip** | 邮件/起飞前 | 焦虑、怕遗漏 | 航班动态、接送机、保险、WiFi |
| **On-trip** | 落地/旅途中 | 兴奋、迷茫 | **LBS定位推荐**、免排队门票、打车券 |
| **Post-trip** | 回程落地 | 分享、回味 | 满意度调查、积分入账、复购券 |
